用Jupyter notebook规划旅行路线

2019-04-10
computer science
Jupyter notebook

【GitHub项目地址】：https://github.com/luochang212/tsp_route

对于在欧洲的小伙伴们，复活节假在这一周就正式开始啦。大家打算怎么计划自己的旅行呢？

身为一个拖延患者，我通常会拖到出发前的最后一个晚上，花上半小时，在网上搜索当地感兴趣的景点 (POI), 然后在Google Maps上为它们点上小星星，以免和它们插肩而过。

每次搜完资料，我的谷歌地图上都攒满了密密麻麻的小星星。此时此刻，望着这些密集的星星，我的偷懒本能适时发动了，不禁发出灵魂之问：如何才能走最少的路，就周游所有景点呢?

我找遍谷歌和百度，都没发现我要的多地点路径规划功能。最接近需求还是谷歌”Add destination” 功能。然而这个功能只是把你点选的地点按最短路径的方式依次连接在一起罢了。可是我们要的不是依次连接，而是最短连接。

那怎么办，自己写一个呗。

适用模型：TSP 模型

首先把问题抽象化，概括要解决的问题。我们的问题是：求从一个地方出发，遍历所有城市，回到起点的最短路径

这其实对应了一个非常经典的早就被数学系的小伙伴们玩腻的问题，辣就是旅行商问题，又名TSP问题。旅行商问题的描述是这样的：一个商人在各个城市之间旅行，要求遍历所有城市并返回到出发点，那么如何规划路线，才能使总路径最短。(这里就不过多解释了，更多信息见维基百科)

解决思路

  • 用googlemaps包获取纬度和经度信息
  • 用OR-Tools包求解TSP问题
  • 最后用gmaps可视化结果

敲下来以后不禁感慨，Google Maps的API真的太好用了，到头来根本不需要自己敲多少代码orz

下面是介绍使用环节

说了这么多，还不快来感受一下定制路线酣畅淋漓的体验（大误）

项目地址 —> 传送门

在体验之前，你需要以下配置：

  1. 你需要一个Jupyter notebook, 当然很多人会直接安装Anaconda3
  2. 你还需要安装这些包：googleplaces, googlemaps, gmaps, ortools. 如果你安了Anaconda, 恭喜，你可以在Anaconda Prompt里安装这些包
  3. 你需要一个Google Maps API key，获取链接: https://developers.google.com/maps/documentation/distance-matrix/start#get-a-key

一位知名不具的哲学家说过，完成配置等于成功的一半。在Jupyter notebook打开本仓库唯一一个ipynb文件，将第一个代码块的所有变量，改成自己的。比如自己的目的地，自己的密码什么的……最有从头到尾运行所有代码块，你就可以得到自己的旅行地图辣~
